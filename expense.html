Project Overview

Expense Tracker is a full-stack MERN application that gives users a simple, secure way to log and manage their personal spending. After signing up or logging in, a user can record an expense with a title, amount, category, date, and optional notes — covering everyday spending like food, transportation, bills, or entertainment — and immediately see it reflected in their dashboard.

I built it to get hands-on with a complete authenticated CRUD application end to end: user accounts, protected routes, a REST API backed by MongoDB, and a responsive frontend that makes adding and reviewing expenses fast rather than tedious.

It's built for anyone who wants a lightweight, no-frills way to track where their money is going — without the complexity of a full budgeting platform.

Problem Statement

Most people don't have a clear picture of where their money actually goes each month. Tracking spending manually in a notebook or spreadsheet is tedious enough that most people give up on it within a few weeks, which means small, recurring expenses go unnoticed until they add up to a real problem.

Existing budgeting apps often solve this by overcomplicating it — bank integrations, subscription tiers, and dense dashboards that create friction rather than removing it. For someone who just wants to log an expense in a few seconds and see a clean history afterward, that complexity gets in the way.

Expense Tracker was built to address exactly that gap: a fast, distraction-free way to add, categorize, and review expenses, with full account security so each user's financial data stays private to them.

Implementation

01

Authentication Module

Handles user sign-up and login, issuing a JWT on successful authentication and protecting all expense routes behind it.

02

Expense Form Module

Captures title, amount, category, date, and notes on the frontend, validates the input, and sends it to the API.

03

API Module

Exposes REST endpoints for creating, reading, updating, and deleting expenses, each scoped to the logged-in user.

04

Database Module

Stores users and their expenses in MongoDB, with each expense document linked back to its owner.

05

Dashboard Module

Fetches and renders the user's expense list, supporting inline edit and delete actions without a full page reload.

Challenges Faced

🔐 Secure Authentication

Implementing JWT-based auth correctly — token storage, expiry, and protected routes — took careful handling to avoid security gaps.

🔄 Real-Time CRUD Sync

Keeping the frontend list in sync after every add, edit, or delete without a full page refresh required careful state management.

🏷️ Category Consistency

Designing a category system flexible enough for varied spending types while staying simple to filter and report on.

📱 Responsive Layout

Making the expense form and dashboard equally usable on small mobile screens as on desktop.
